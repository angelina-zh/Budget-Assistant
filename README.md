# Budget-Assistant
Программа хранит ежедневные расходы пользователя за месяц. Пользователь может добавить расход (день, сумма, категория) или запросить сумму расходов за период с дня A по день B. Для ускорения ответов на такие запросы построен массив префиксных сумм (накопительная сумма расходов по дням). Ответ на запрос — за O(1). 
Реализован
линейный поиск дня с максимальным расходом. Сортировка вставками для категорий по
сумме трат. Стек для отмены последнего добавленного расхода. Дерево для хранения трат.
Структура файлов проекта:
bst.py – реализация бинарного дерева поиска для хранения транзакций (работа Жильцовой
Ангелины).
sort.py – реализация сортировки вставками (работа Жильцовой Ангелины).
budget_assistant.py – основной класс BudgetAssistant, содержащий всю логику учёта
расходов, префиксные суммы и стек отмен (работа Янчук Юноны).
main.py – главный модуль, запускает программу и реализует консольное меню (работа
Янчук Юноны).

Используемые структуры данных и алгоритмы:
Префиксные суммы используются для быстрых запросов суммы расходов за любой
период. Программа хранит массив накопленных сумм по дням, поэтому чтобы узнать,
сколько потрачено с a по b день, нужно просто вычесть два числа. Это работает за
постоянное время O(1).
Линейный поиск применяется для нахождения дня с максимальным расходом. Программа
просто проходит по всем дням месяца и запоминает день с самой большой суммой.
Сортировка вставками реализована вручную для сортировки категорий расходов.
Категории выводятся в порядке убывания сумм: сначала те, на которые потратили больше
всего. Алгоритм работает так: берем очередную категорию и вставляем ее в уже
отсортированную часть списка на правильное место.
Стек используется для отмены последней операции. Каждый раз при добавлении расхода
программа сохраняет информацию о нем в стеке. Если пользователь передумал, данные
извлекаются из стека и все возвращается обратно.
Бинарное дерево поиска хранит все транзакции. Каждая транзакция получает уникальный
номер, который становится ключом в дереве. Дерево позволяет быстро добавлять новые
расходы, удалять их при отмене и выводить все транзакции по порядку. Для вывода
используется рекурсивный симметричный обход — алгоритм сначала проходит по левым
потомкам, потом обрабатывает текущий узел, потом идёт в правых потомков. Так
транзакции выводятся в порядке возрастания их номеров.

- вывод категорий, отсортированных по сумме.
- отмена последней операции.
- вывод всех транзакций с помощью обхода бинарного дерева.

вставками.
